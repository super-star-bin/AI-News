# 来源分级规则

发布前优先使用 S/A/B 级来源支撑结论。C/D 级来源可以用于发现线索，但不能单独作为事实依据。

## S 级

最高可信度，可作为事实依据。

- 官方博客
- 官方产品文档
- Release Notes
- 论文原文
- GitHub 仓库、Issue、PR、Release
- 官方 API 文档

用途：事实核验、功能说明、发布时间、价格和可用性确认。

## A 级

高可信度，可作为观点、趋势和背景依据。

- 研究者公开发言
- 创始人或核心员工公开发言
- 会议演讲
- 官方访谈
- 公司技术报告

用途：趋势判断、产品方向、技术路线。

## B 级

中高可信度，适合补充商业和行业背景。

- Reuters
- Financial Times
- TechCrunch
- The Verge
- MIT Technology Review
- Bloomberg
- Wired

用途：行业动态、融资、商业化、竞争格局。

## C 级

中可信度，只能作为线索或用户反馈来源。

- Hacker News
- Reddit
- Product Hunt
- Discord 社区
- 开发者论坛
- 用户评论区

用途：发现讨论热度、痛点和真实使用反馈。

## D 级

低可信度，不能单独作为事实依据。

- 二手中文搬运
- 短视频爆料
- 未标来源截图
- 营销号总结
- 群聊截图

用途：只做选题线索。发布前必须回到 S/A/B 级来源核验。
