# 事实核验：AI 编程 Agent 进入开源供应链

topic_id: T20260627-004  
草稿：`drafts/xhs_2026-06-27_ai-coding-agents-open-source.md`  
来源：arXiv 论文  
来源等级：S  
原始链接：https://arxiv.org/abs/2606.24429

## 核心结论

可写：AI 编程 Agent 活动已经在开源仓库中形成可检测痕迹，只看 bot 账号或 PR 会漏掉大量活动。  
不可写：所有开源项目都已经被 AI Agent 大规模接管，或 Agent 贡献一定低质量/高风险。

## 已核验事实

| 事实 | 来源 | 来源等级 | 风险 |
| --- | --- | --- | --- |
| arXiv 论文标题为 `Detecting AI Coding Agents in Open Source`。 | arXiv | S | 低 |
| 论文摘要称研究对象是开源开发中 AI coding agents 的采用情况。 | arXiv | S | 中 |
| 论文摘要称研究使用超过 1.8 亿个仓库作为规模背景。 | arXiv | S | 中 |
| 论文摘要称 bot-account 方法只识别 28,154 个 Claude Code 提交，占 3.3%。 | arXiv | S | 中 |
| 论文摘要称 commit-attributed detection 显示 2024-12 到 2026-04 期间每月超过 320,000 次提交。 | arXiv | S | 中 |
| 论文摘要称 Claude Code 在 17,295 个项目中产生 886,122 次提交。 | arXiv | S | 中 |
| 论文摘要称 pull request census misses 79% of contributions。 | arXiv | S | 中 |

## 账号判断

- 对开源维护者和工程团队来说，重点不是“AI 写代码好不好”，而是审查流程不能只看显性 bot 或 PR。
- 对普通职场人来说，这说明 AI Agent 已经从工具体验进入软件供应链和协作规则。

## 限制与不确定

- 论文是预印本。
- 检测方法可能存在误判或漏判。
- 论文讨论的是检测和规模，不等同于评价所有 Agent 代码质量。
- 不把数字泛化到所有平台、所有语言或所有开源项目。
