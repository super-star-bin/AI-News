# 数据主库说明

本目录是小红书 AI 账号运营系统的本地轻量主库。验证期不使用 Notion 和飞书，所有结构化数据先写入 CSV。

## 文件用途

- `topics.csv`：候选选题主库。记录来源、栏目、评分、事实核验、实测状态、发布状态和草稿路径。
- `posts.csv`：已发布笔记数据。记录小红书链接、浏览、点赞、收藏、评论、关注转化和 D1/D7 复盘。
- `user_needs.csv`：用户需求库。记录评论、私信和高频问题，用于资料包、模板、课程和咨询方向判断。

## ID 命名规则

- 选题 ID：`TYYYYMMDD-001`，例如 `T20260625-001`。
- 发布 ID：`PYYYYMMDD-001`，例如 `P20260625-001`。
- 需求 ID：`NYYYYMMDD-001`，例如 `N20260625-001`。

## 状态枚举

### 发布状态

- `候选`
- `写作中`
- `待发布`
- `已发布`
- `放弃`

### 事实核验

- `待核验`
- `已核验`
- `证据不足`

### 实测状态

- `不需要`
- `待实测`
- `已实测`
- `失败`

## 使用规则

- 不要删除 CSV 表头。
- 不要手动复用 ID。
- 同一个 URL 不重复入库。
- 发布后必须用 `topic_id` 关联 `posts.csv`。
- 周复盘读取 `topics.csv`、`posts.csv` 和 `user_needs.csv`。
