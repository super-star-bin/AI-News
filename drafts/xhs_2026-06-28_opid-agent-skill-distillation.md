# 小红书草稿：OPID 值得收藏，但不适合普通人直接跑

## 推荐标题

OPID 值得收藏，但不适合普通人直接跑

## 发布栏目

开源项目拆解

## 正文

今天拆一个偏研究向的开源项目：`jinyangwu/OPID`。

OPID 的论文标题是 `On-Policy Skill Distillation for Efficient Long-Horizon LLM Agent Learning`。中文可以理解为：从模型实际执行过的任务轨迹里，提炼出可复用技能，用来训练更会做长任务的 LLM Agent。

我的判断：OPID 值得收藏，但不适合普通人直接跑。它不是“装上就能帮你办公”的 Agent 工具，而是一个学习 Agent 训练方法的项目。

来源事实先讲清楚。GitHub 仓库公开代码，README 写明这是 OPID 的官方实现，license 显示为 MIT。论文摘要称，OPID 会把 completed on-policy trajectories，也就是模型实际完成任务后的轨迹，转成 hierarchical hindsight skills，中文可以理解为“分层的事后技能”。论文还称它在 ALFWorld、Search-QA、WebShop 等长程任务上提升了表现。

为什么值得看？

过去很多 Agent 失败，不是因为不会写一句回答，而是长任务中容易走偏、重复试错、不会复用成功经验。OPID 这类方法的意义在于：让 Agent 从自己跑过的成功轨迹里提炼“技能”，而不是每次都从零规划。

两个具体场景可以理解。

第一，做网页购物 Agent，不只是点击按钮，还要学会“先找商品、再比参数、再加入购物车”这类阶段性技能。

第二，做问答搜索 Agent，不只是检索一次，而是学会在搜索、筛选、验证之间循环。

但上手门槛很高。README 里涉及 Python 3.12、vLLM、flash-attn、OpenAI-compatible endpoint 和多个评测环境。本账号没有本地安装实测，也不能说它可以直接用于你的工作流。

适合谁：Agent 研究者、AI 工程师、想了解长程任务训练的人。

不适合谁：只想找一个能直接帮你写日报、做表格、发内容的普通用户。

一句话总结：OPID 的价值不是“现在就拿来用”，而是提醒我们，Agent 真正难的是把成功经验沉淀成可复用技能。

评论区可以聊：你更想看开源项目的上手教程，还是先看这种“值不值得收藏”的拆解？
