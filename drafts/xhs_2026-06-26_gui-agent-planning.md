## 正文

今天的 AI Agent 应用观察，来自一篇 2026 年 6 月 25 日提交到 arXiv 的论文：`Empowering GUI Agents via Autonomous Experience Exploration and Hindsight Experience Utilization for Task Planning`。

中文可以理解为：通过自主经验探索和事后经验利用，提升 GUI Agent 的任务规划能力。

我的判断：这篇论文值得看，因为它提醒我们：GUI Agent 真正难的不是会点按钮，而是能不能把复杂目标拆成计划，并在新网站、新异常里继续完成任务。

GUI Agent 可以理解为能在网页或软件界面里操作任务的智能体。未来它可能帮你填表、查订单、上传材料、整理后台。但页面、任务、输入都会变，所以不能只靠固定规则。

两个场景最能说明问题。

第一个是报销。它不能只会点“新建报销单”，还要选类型、上传发票、识别金额、校验预算，遇到缺字段时停下来问人。

第二个是客服后台。它要查身份、核对订单、判断问题类型，再根据权限决定能不能改状态。最危险的不是没点到按钮，而是点错、越权，或信息不足时继续执行。

来源事实：arXiv 摘要写到，多模态网页 Agent 可以帮助人类处理重复 GUI 任务，而 task planning，也就是任务规划，是把复杂任务拆成动作的关键。摘要还说，小型开源 MLLM，也就是多模态大语言模型，成本和隐私上有优势，但规划弱、跨网站泛化有限。

论文提出 PEEU，大意是让 Agent 自主探索环境，再利用 hindsight experience，也就是“事后经验”，合成高层训练数据。最值得记住的是：掌握低层原子技能，不保证高层规划能力；高层任务训练带来更强的 OOD generalization，也就是遇到新任务、新网站时仍能泛化完成。

翻成普通话：会点按钮，不等于会办事。

评估 GUI Agent，不要只看演示视频。至少问 5 个问题：换网站还能完成吗？失败会不会重试？能解释计划吗？每步有没有日志？权限边界能不能控制？

限制也要保留：摘要称其 7B 模型在特定 benchmark 上达到 30.6% accuracy，并超过 Qwen2.5-VL-32B。但这不能写成“小模型全面超过大模型”。更稳妥的结论是：在这篇论文设置下，高层任务经验对规划能力很关键。

一句话总结：Agent 不是自动点击器。真正值得关注的，是它能不能计划、复盘、处理异常。
